QUADRUJUGGLE — NEXT STEPS (not started yet)
--------------------------------------------
The main project: train Go1 to balance a ping-pong ball on a back-mounted paddle.
README at: ~/Research/QuadruJuggle/README.md
No code written yet — only README + git scaffolding.

Phase 1 plan (privileged state PPO):
  1. Create ball + paddle USD assets (sphere primitive + flat box, fixed joint to trunk)
  2. Build custom IsaacLab environment (new extension in QuadruJuggle repo):
     - Scene: Go1 + paddle + ball
     - Obs: ball pos/vel relative to paddle center, robot proprioception (48 + 6 = 54 dims)
     - Reward: exp(-k * ||ball_xy - paddle_center_xy||) + penalty on body velocity
     - Termination: ball exits paddle boundary radius
  3. PPO training config (copy flat Go1 config, wire to new env)
  4. Static balance milestone first (robot standing still, ball on paddle)

Phase 2 (teacher-student, later):
  - Egocentric camera on base link looking down at paddle
  - Student policy: CNN/DINOv2 encoder → latent replacing privileged ball obs
  - DAgger/BC distillation from Phase 1 teacher

Key papers for Phase 2:
  - Teacher-Student: https://arxiv.org/abs/2012.01052
  - DAgger: https://arxiv.org/abs/1011.0686

For QuadruJuggle, anticipate needing:
  - Denser reward (ball never touches the ground = sparse → use exponential kernel)
  - Tighter termination (ball off paddle)
  - Possibly curriculum: start ball at rest, then add perturbations
  - May need to freeze locomotion policy and train only balance layer (hierarchical)
    OR train end-to-end from scratch with combined reward
